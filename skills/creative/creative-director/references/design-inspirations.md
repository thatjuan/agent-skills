## How to Use This Catalog

This catalog provides access to real-world design systems as raw reference material. The creative director has full autonomy over concept development — these are starting points for inspiration, not templates to replicate.

**Useful for:**
- Studying how leading brands in the same industry approach visual language
- Identifying color palette strategies, typography pairings, and spacing philosophies that resonate with specific audiences
- Understanding what "premium", "approachable", "technical", or "playful" looks like in practice across different brands
- Finding unexpected inspiration by looking outside the client's industry
- Grounding abstract design concepts in concrete, proven examples

**The creative process remains original.** Fetched design systems inform concept development the way a creative director's visual library informs their thinking — as absorbed vocabulary, not as sources to copy.

## Fetching a DESIGN.md

Each design system is available as a structured markdown document:

```
https://getdesign.md/{company}/design-md
```

Use `WebFetch` to retrieve any design system during concept development. Fetch 2-4 relevant systems as inspiration — both from the client's industry and from unexpected adjacent industries.

## DESIGN.md Structure

Each fetched document contains 9 standardized sections:

| Section | What It Contains |
|---------|-----------------|
| Visual Themes & Atmosphere | Descriptive keywords and aesthetic qualities |
| Color Palettes | Semantic color naming with hex codes |
| Typography | Font families, hierarchies, text styling |
| Component Styling | UI elements with state variations |
| Layout Spacing | Spacing principles and grid systems |
| Depth & Shadows | Shadow systems for elevation |
| Design Guardrails | Usage constraints and best practices |
| Responsive Breakpoints | Mobile, tablet, desktop specifications |
| Agent Prompts | AI-readable design instructions |
